## 1. 프로젝트 개요

- **프로젝트 이름**:  Vintly
- **한 줄 설명**: 빈티지 샵 및 제품 정보를 공유하고 소통하는 커뮤니티 모바일 앱
- **간단한 소개**:  
  - 빈티지를 좋아하는 사람들만에 커뮤니티가 없어 기획하였다.
  - 빈티지 매장에 대한 위치, 사진 정보를 제공하고 좋아요 및 댓글을 쓸 수 있다.
  - 추후 게시글 기능을 추가하여 빈티지 제품(의류 등)에 대한 정보를 공유하고 게시글과 댓글로 소통하는 커뮤니티 앱으로 만들 생각이다.
  

## 2. 타겟 유저 & 사용 시나리오

- **타겟 유저**:
  - 연령대: 20~40대

  - 특징(직업, 관심사 등): 
     - 빈티지 패션/제품에 관심이 있는 누구나

  - 예: “나는 **빈티지 샵 사장으**로서, **내 빈티지 샵 관심드롤 확인하기 위해**사용한다.”

    - 네트워크: (예: `dio`, `http`)

    - 로컬 저장소: (예: `shared_preferences`, `hive` 등)

- **아키텍처 패턴**:
  - 예: MVVM + Repository 패턴
  - 예: Feature-based 폴더 구조 (`features/auth`, `features/home` 등)

- **API 인증 및 401 처리**:
  - **규칙: 모든 API에서 401이 뜨면 우선 reissue를 시도한다.**
  - access 토큰이 붙는 인증 API 호출 시, 서버가 401을 반환하면 클라이언트는 먼저 refresh 토큰으로 `/api/v1/auth/reissue`를 호출해 새 access·refresh를 받고, **해당 API 요청을 새 토큰으로 한 번만 재시도**한다.
  - reissue 실패 또는 재시도 후에도 401이면 "로그인 정보가 만료되었습니다" 등으로 안내하고 다시 로그인 유도.
  - 구현: 인증이 필요한 API는 `lib/shared/api/authenticated_api.dart`의 `getJsonWithAuth`, `postJsonWithAuth`를 사용하면 401 시 자동으로 reissue 후 한 번 재시도된다.
